1 INTRODUCCIÓN

El presente documento de requisitos de software (SRS) del sistema surge de un estudio y analisis previo hacia el organismo a el cual se piensa hacer el producto, se espera que el proceso de elaboracion se hara para que el cliente se sienta satisfecho.

1.1 Proposito

- Documentar y estructurar las peticiones acordadas del cliente.
- Mostrar de una manera facil y entendible a los usuarios finales que es lo que el cliente desea.

1.2 Alcance

- Identificacion del producto de software "SCL"
- Objeitvos del sistema:
- Permitir la gestion la consulta de libros.
- Mostrar el Lugar donde se encuentran los libros.
- Mostrar la informacion importante de los libros.

1.4 Definiciones y acronimos

1.4.1 Definiciones

-ACTUALIZACION.- Insertar, eliminar, modificar los registros de los usuarios (Profesores).
-ALMACENAMIENTO- En relación con ordenadores o computadoras, cualquier dispositivo capaz de almacenar información procedente de un sistema informático.
-ANDROID.- Es un sistema operativo basado en el núcleo Linux. Fue diseñado principalmente para dispositivos móviles con pantalla táctil, como teléfonos inteligentes o tablets; y también para relojes inteligentes, televisores y automóviles.
-BASE DE DATOS.- Cualquier conjunto de datos organizados para su almacenamiento en la memoria de un ordenador o computadora, diseñado para facilitar su mantenimiento y
acceso de una forma estándar. La información se organiza en campos y registros. Uncampo se refiere a un tipo o atributo de información, y un registro, a toda la información sobre un individuo.
-CONEXION.- Comunicación entre dos entes que tienen características similares de comunicación.
Interfaz.- Medio que permite la comunicación entre el usuario y el sistema.
-DISPOSITIVOS MOVILES.- Se puede definir como un aparato de pequeño tamaño, con algunas capacidades de procesamiento, con conexión permanente o intermitente a una red, con memoria limitada, que ha sido diseñado específicamente para una función, pero que puede llevar a cabo otras funciones más generales.
-HOST/HOSTING- Es un ordenador que funciona como el punto de inicio y final de las transferencias de datos. Más comúnmente descrito como el lugar donde reside un sitio web. Un host de Internet tiene una dirección de Internet única (dirección IP) y un nombre de dominio único o nombre de host.
-INTERNET- interconexión de redes informáticas que permite a los ordenadores o computadoras conectadas comunicarse directamente, es decir, cada ordenador de la red puede conectarse a cualquier otro ordenador de la red. El término suele referirse a una interconexión en particular, de carácter planetario y abierto al público, que conecta
redes informáticas de organismos oficiales, educativos y empresariales.
-LOGIN.- Nombre o alias que se le da a una persona para permitirle el acceso al sistema siempre y cuando estén registrados.
-PASSWORD.- Contraseña o clave para autentificar el ingreso a un lugar o sitio.
-SERVIDOR.- Computadora conectada a una red que pone sus recursos a disposición del resto de los integrantes de la red. Suele utilizarse para mantener datos centralizados o para gestionar recursos compartidos.
-SERVICIOS WEB.- Un servicio web es una interfaz de software que describe un conjunto de operaciones a las cuales se puede acceder por la red a través de mensajería XML estandarizada. Usa protocolos basados en el lenguaje XML con el objetivo de describir una operación para ejecutar o datos para intercambiar con otro servicio web.
- SITIO WEB.- Se canaliza a través del URL o identificador único de cada página de contenidos. Este sistema permite a los usuarios iniciar una solicitud de trámite y a los funcionarios del Agua Potable atender las solicitudes e ingresar datos de las inspecciones realizadas.

1.4.2 Acrónimos

-DBMS.- Sistema de Administración de base de datos, El software que permite la creación, administración e implementación de la base de datos.
-SCL. Sistema de consulta de libros
-JSON.- (JavaScript Object Notation) es un formato para el intercambios de datos, básicamente JSON describe los datos con una sintaxis dedicada que se usa para identificar y gestionar los datos.
-PHP es un lenguaje de programación de uso general de código del lado del servidor originalmente diseñado para el desarrollo web de contenido dinámico.
-SRS.- Acrónimo de Software Requeriments Specifications (Especificación de
Requerimientos de Software)

El sistema se va a desarrollar de manera independiente y tendra un diseño modular donde contara con 2 secciones, los cuales son: El del administrador, se encargara de gestionar los libros que se encuentre en la organizacion y el del usuario final, es el que consultara el libro y podra apreciar en que lugar esta ubicado.

2.4 Restricciones

El sistema sera desarrolado en PHP un Lenguaje que sera usado para crear los servicios Web, el motor de base de datos es MySql de tecnologia Relacional,el DBMS que se usara es PhpMyAdmi, se usara Java para el manejo de datos y XML para la interfaz en la aplicacion Android, se usaran datos Json para que sean mas sencillo manejarlos.

2.5 Suposiciones y dependencias

2.5.1 Dependencias

Para usuarios finales se debe tener un dispositivo movil con un sistema operativo Android 4.0 como minimo, tambien se debe tener una conexion de datos para que se pueda usar la aplicacion.

Para el administrador debe tener un navegador reciente para evitar problema de compatibilidad, tambien una conexion estable para que no se pierda los datos al gestionarse.

R1: Permitir la autenticacion del administrador y usuarios.

R4: Permitir la reserva de un libro que el usuario desee.

3.1 Requisitos Comunes de los interfaces

3.1.1 Interfaces de usuario

Las interfaces de usuario estan relacionadas a pantallas de dispositivos moviles como tambien dispositivos de mesa estos mediante Formularios y listas para que se pueda mostrar de una forma comoda al usuario final. Las interacciones con la aplicaciones se haran por medio tactil (Caso de consulta) y por medio del Teclado y el Mouse(Caso del administrador).

Las interfaces que se van a crear incluiran:
- Botones
- Menu
- Alertas
- Listas
- Formulario de autenticacion.

3.1.2 Interfaces de hardware

- La Pantalla tactil del dispositivo Movil: La aplicacion Android debera interactuar con el usuario final y mostrar la informacion de los libros.

- La pantalla del monitor: El software mostrar los formularios para que el Adminsitrador pueda gestionar la informacion de los libros.

- Raton: El software debera interactuar con las acciones del Raton para poder validar la informacion escrita.

- Teclado: El software debera interactuar con las pulsaciones del teclado para poder llenar los campos de los formularios para el ingreso de estos datos a los sectores que deben ir en la base de datos.

3.1.3 Interface de software

Para hacer uso de la aplicacion se usara un dispositivo movil con sistema operativo Android minimo en su version 4.0.

3.3 Requisitos no funcionales

3.3.1 Requisitos de rendimiento

El dispositivo movil debera contar con una conexion de datos estable para tener unos tiempos de respuesta minimos.

El numero de usuarios que interactuaran simultanineamente con nuestro sistema esta en un rango de 50-100 como maximo.

El servidor de base de datos, debera contar con un personal tecnico listo para cualquier eventualidad.

3.3.2 Seguridad

La seguridad del sistema es por:

- Uso de contraseñas que contenga : un minimo de 6 caracteres, que contenga como minimo caracter numerico, caracteres y que no tenga espacios.

3.3.3 Fiabilidad

Es un factor que le generara mas facilidad a los usuario que se encuentran en la organizacion y usando la tecnologia de dispositivos moviles que es de uso cotidiano en esta epoca.

3.3.4 Disponibilidad

El sistema ha sido realizado tomando en cuenta el horario de la organizacion y el uso del establecimiento y estara disponible el 100% del tiempo del dia.

3.3.5 Mantenibilidad

No se necesitara en esta aplicacion.

3.3.6 Portabilidad

Una de las ventajas de utilizar lenguajes basicos y de software libre estamos garantizando que sea multiplataforma y una base de datos fiable para la organizacion.
